# AVEVA 타입 네임스페이스 확인 (빌드 오류 CS0246 해결용)

빌드 시 `Standalone`, `PdmsMessage`, `TypeFilter`, `DBElementCollection`, `DbElementTypeInstance`,
`CurrentElement` 같은 타입을 못 찾는 오류(CS0246)가 나면, 이 타입들이 **Marine(OH12.1)의 어느
네임스페이스·어셈블리에 있는지** 확인해야 합니다. 아래 명령으로 한 번에 뽑아 전달해 주세요.

---

## 0. 가장 쉬운 방법 — 더블클릭 (권장)

붙여넣기/줄바꿈/비트수 문제를 모두 피하려면, 압축 안의 **`check-types.cmd` 더블클릭**.
- 32비트 PowerShell로 실행 + 의존성 없는 메타데이터 방식이라 결과가 확실히 나옵니다.
- 결과가 화면에 뜨고, **바탕화면 `aveva_types.txt`** 에도 저장됩니다 → 그 파일을 보내주세요.
- 만약 "NO TYPES FOUND" 가 뜨면 화면 아래 DLL 목록이라도 보내주세요.

> 아래 1번(붙여넣기)이 안 먹혔던 이유: 보통 **Marine DLL이 32비트라 64비트 PowerShell에서 로드 실패** →
> catch 가 삼켜서 아무것도 안 나옴. `check-types.cmd` 가 이를 해결합니다.

---

## 1. (수동) PowerShell에 그대로 붙여넣기

```powershell
$bin = "C:\AVEVA\Marine\OH12.1.SP5"
$wanted = "DbElement","DbElementType","DbAttribute","DbAttributeInstance","TypeFilter","DBElementCollection","DbElementTypeInstance","CurrentElement","PdmsMessage","Project","MDB","Standalone"
$out = @()
foreach($dll in Get-ChildItem $bin -Filter "Aveva*.dll"){
  try{
    $asm=[Reflection.Assembly]::LoadFrom($dll.FullName)
    try{ $ts=$asm.GetTypes() }catch{ $ts=$_.Exception.Types | Where-Object {$_} }
    foreach($t in $ts){ if($t -and ($wanted -contains $t.Name)){ $out += ("{0,-22} ns={1,-34} dll={2}" -f $t.Name,$t.Namespace,$dll.Name) } }
  }catch{}
}
$out = $out | Sort-Object -Unique
$out
$out | Set-Content "$env:USERPROFILE\Desktop\aveva_types.txt"
Write-Host "`n(결과가 바탕화면 aveva_types.txt 에도 저장됨)"
```

---

## 2. 결과 보내기

- 위 명령이 출력하는 줄들을 **그대로 복사**해서 전달하거나,
- 바탕화면에 생성된 **`aveva_types.txt`** 파일을 보내주세요.

예상 출력 형태:
```
DbElement              ns=Aveva.Pdms.Database            dll=Aveva.Pdms.Database.dll
TypeFilter             ns=Aveva.Pdms.Database.????       dll=Aveva.Pdms.Database.dll
DBElementCollection    ns=Aveva.Pdms.????                dll=...
PdmsMessage            ns=Aveva.Pdms.????                dll=...
CurrentElement         ns=Aveva.Pdms.????                dll=...
Standalone             ns=Aveva.Pdms.Standalone          dll=Aveva.Pdms.Standalone.dll
...
```

이 목록을 받으면 프로젝트의 모든 `using` 과 어셈블리 참조를 **실제 값으로 한 번에 교체**해서
빌드가 통과하도록 고쳐 드립니다.

## 3. (참고) 아무 줄도 안 나올 때

- DLL 로드가 막히는 환경일 수 있습니다. 그땐 아래로 **DLL 목록만** 먼저 보내주세요:
  ```powershell
  Get-ChildItem "C:\AVEVA\Marine\OH12.1.SP5" -Filter "Aveva.Pdms.*.dll" | Select-Object -ExpandProperty Name | Sort-Object
  ```
- 또는 ILSpy / dotPeek 같은 디컴파일러로 `Aveva.Pdms.Standalone.dll` 등을 열어
  `Standalone`, `PdmsMessage`, `TypeFilter` 의 네임스페이스를 확인해 알려주셔도 됩니다.
